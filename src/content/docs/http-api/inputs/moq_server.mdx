An input type that allows Smelter to receive streams over Media over QUIC (MoQ).

Smelter listens for incoming MoQ streams after the input is registered, you can do it by sending [`register-input` request](/http-api/routes#register-input).

A publishing client must announce the configured `broadcast_path` for the input to receive media.

## Usage

To use MoQ server input you must register it first. You can do it by sending a request like this:

<CollapsibleDetails summaryTitle='Example request' open>

```http
POST: /api/input/:input_id/register
Content-Type: application/json

{
  "type": "moq_server",
  "broadcast_path": "example/broadcast"
}
```
</CollapsibleDetails>

See [HTTP Routes](/http-api/routes#inputs-configuration) documentation to learn more about managing inputs.

Once the MoQ server input is registered and Smelter is listening on the configured broadcast path, you can push a stream to it using any MoQ-compatible client.

## Reference

<CollapsibleDetails summaryTitle='Type definition' open>
```tsx
type MoqServerInput = {

  decoder_map?: DecoderMap;
}
```
</CollapsibleDetails>

Parameters for registering a MoQ server input.

## Properties

### broadcast_path
The MoQ broadcast path that Smelter listens on for incoming streams.
A publishing client must announce this broadcast path for the input to receive media.

- **Type**: `string`

---

### required
Determines if the input stream is essential for output frame production. If set to true and the stream is delayed, Smelter will postpone output frames until the stream is received.
- **Type**: `bool`
- **Default value**: `false`

---

### decoder_map
Assigns which decoder should be used for media encoded with a specific codec.

- **Type**: [`DecoderMap`](#decodermap)


## DecoderMap
Maps codecs to the provided decoders.

<CollapsibleDetails summaryTitle='Type definitions' open>
```tsx
type DecoderMap = {
  h264?: 'ffmpeg_h264' | 'vulkan_h264';
};
```
</CollapsibleDetails>

### Properties

#### h264
H264 decoder configuration.

- **Type**: `'ffmpeg_h264' | 'vulkan_h264'`
- **Default value**: If available `vulkan_h264` will be used, otherwise `ffmpeg_h264`
- **Supported values**:
  - `"ffmpeg_h264"` - Software H264 decoder based on FFmpeg.
  - `"vulkan_h264"` (<Badge text="Required feature: vk-video" variant="note" />) - Hardware decoder. Requires GPU that supports Vulkan Video decoding.
