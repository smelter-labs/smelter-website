A component for rendering images:
- From external URLs
- From the server hosting a Smelter instance

<Aside> To use this component, you need to first register the image with matching `image_id` using [`register image`](/http-api/routes#register-image) request. </Aside>

## Example

Before using an Image in a stream, it must first be registered via an HTTP request. Use the following request to register a PNG image:

<CollapsibleDetails summaryTitle='Register PNG image' open>
```http
POST: /api/image/example_png/register
Content-Type: application/json

{
  "asset_type": "png",
  "url": "https://example.com/image.png"
}
```
</CollapsibleDetails>

Once this request is successfully processed, the Image will be available for use in the output stream. You can reference a specific Image by its unique `image_id` (in this example `example_png`)

<CollapsibleDetails summaryTitle='Update output' open>
```http {10-11}
POST: /api/output/example_output/update
Content-Type: application/json

{
  "video": {
    "root": {
      "type": "rescaler",
      "mode": "fit",
      "child": {
        "type": "image",
        "image_id": "example_png"
      }
    }
  }
}
```
</CollapsibleDetails>

<CollapsibleDetails summaryTitle='Type definitions'>
```tsx
type Image = {

```
</CollapsibleDetails>


## Properties

### id
ID of a component.

- **Type**: `string`

---

### image_id
ID of an image. It identifies an image registered using a [register image](/http-api/routes#register-image) request.

- **Type**: `string`
