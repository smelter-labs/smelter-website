An input type that allows Smelter to receive streams over RTMP.

Smelter exposes an RTMP endpoint after the input is registered, you can do it by `Smelter.registerInput`.

The RTMP connection is configured using the following parameters:
- The Smelter server address
- The configured RTMP port - defaults to `1935`, configurable via [`SMELTER_RTMP_SERVER_PORT`](/deployment/configuration#smelter_rtmp_server_port) 
- The registered `app`
- The registered `streamKey`

Most RTMP clients accept connection parameters in form of URL.\
Format: `rtmp[s]://<smelter_ip>:<port>/<app>/<stream_key>`

<Aside>
To enable RTMPS (RTMP over TLS/SSL), configure the following environment variables:
- [`SMELTER_RTMP_TLS_CERT_FILE`](/deployment/configuration#smelter_rtmp_tls_cert_file) — path to the TLS certificate file
- [`SMELTER_RTMP_TLS_KEY_FILE`](/deployment/configuration#smelter_rtmp_tls_key_file) — path to the TLS private key file
</Aside>

## Usage
```tsx title=rtmpServerInputExample.tsx
import Smelter from "@swmansion/smelter-node";

async function run() {
  const smelter = new Smelter();
  await smelter.init();
  await smelter.registerInput("example", {
      type: "rtmp_server",
      app: "myapp",
      streamKey: "mykey",
  });
}
void run();
```

Once the RTMP server input is registered and Smelter is listening on the configured URL, you can push a stream to it using any RTMP-compatible broadcaster (f.e. OBS, FFmpeg).

Parameters for registering an RTMP server input.

## Properties

### app
The RTMP application name.
This is the first path segment of the RTMP stream URL that Smelter listens on for incoming streams.
Format: `rtmp://<ip_address>:<port>/<app>/<stream_key`

- **Type**: `string`

---

### streamKey
The RTMP stream key.
This is the second path segment of the RTMP stream URL that Smelter listens on for incoming streams.
Format: `rtmp://<ip_address>:<port>/<app>/<stream_key`

- **Type**: `string`

---

### required
Determines if the input stream is essential for output frame production. If set to true and the stream is delayed, Smelter will postpone output frames until the stream is received.

- **Type**: `boolean`
- **Default value**: `false`

---

### decoderMap
Assigns which decoder should be used for media encoded with a specific codec.

- **Type**: [`DecoderMap`](#decodermap)


## DecoderMap
Maps codecs to the provided decoders.

<CollapsibleDetails summaryTitle='Type definitions' open>
```tsx
type DecoderMap = {

};
```
</CollapsibleDetails>

### Properties

#### h264
H264 decoder configuration.

- **Type**: `'ffmpeg_h264' | 'vulkan_h264'`
- **Default value**: If available `vulkan_h264` will be used, otherwise `ffmpeg_h264`
- **Supported values**:
  - `"ffmpeg_h264"` - Software H264 decoder based on FFmpeg.
  - `"vulkan_h264"` (<Badge text="Required feature: vk-video" variant="note" />) - Hardware decoder. Requires GPU that supports Vulkan Video decoding.
