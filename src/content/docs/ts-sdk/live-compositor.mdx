The primary element required to start using the Live Compositor TypeScript SDK is the `LiveCompositor` class. This class is crucial for initializing and managing the Live Compositor server instance.

## new LiveCompositor()
<Code code={`new LiveCompositor(manager?: CompositorManager)`} lang="tsx" />

Creates new compositor configuration. You have to call init first before this object can be used.


### Parameters

#### manager
Configures how the client code will connect and manage `LiveCompositor`. 

- **Type**: [`CompositorManager`]()
<br/>
You have two main configuration options:
- **Default Configuration:** Use `LocallySpawnedInstance` from `@live-compositor/node` to automatically download the necessary LiveCompositor binaries and launch the server on your local machine.
- **Existing Server Configuration:** Choose `ExistingInstance` from `@live-compositor/node` if you need to connect to a compositor instance that is already running.

## init()
<Code code={`LiveCompositor.init(): Promise<void>`} lang="tsx" />

Initialize the LiveCompositor instance. This might spawn a new instance or just establish a connection, depending on the `manager` used while creating LiveCompositor instance.

After `init()` you can start connecting inputs/outputs or register other elements. However, no output stream will be produced until `start()` method is called.

## start()
<Code code={`LiveCompositor.start(): Promise<void>`} lang="tsx" />

Starts the processing pipeline. Any previously registered output will start producing the video/audio stream.
