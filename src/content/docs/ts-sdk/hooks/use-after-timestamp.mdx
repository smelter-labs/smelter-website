`useAfterTimestmp` returns boolean that represents whether specified [timestamp](/fundamentals/glossary#timestamp) already passed.

For [offline processing](/fundamentals/glossary#offline-processing), you can't rely on real time clock to measure time. This hook replaces patterns
where you would use `useEffect` + `setTimeout` in a live processing case.

<Aside>
This hook works both for live and offline processing, but it is primarily intended for
offline processing.
</Aside>

## Reference

```tsx
function useAfterTimestamp(timestampMs: number): boolean
```

## Arguments

### timestampMs

[Timestamp](/fundamentals/glossary#timestamp) in milliseconds.

- **Type**: `number`

## Returns

`true` if the time defined by the timestamp already passed, `false` otherwise.

- **Type**: `boolean`
